# 📌 Sistema de Gestión de Incidencias TI (RIHO)

## 📖 Descripción del Proyecto
Este proyecto consiste en el desarrollo de un **software de gestión de incidencias** para el área de Tecnologías de la Información (TI) de la empresa **Hernán Ocazionez S.A.S.**  

La solución busca reemplazar los procesos manuales actuales mediante una **aplicación web centralizada**, permitiendo registrar, asignar, monitorear y cerrar incidencias de forma eficiente.

---

## 🎯 Objetivo General
Implementar una mesa de ayuda interna que mejore la gestión de incidentes y solicitudes del área de sistemas.

---

## 🚀 Funcionalidades Principales
- 📌 Registro de incidencias
- 🔄 Modificación de tickets
- 👨‍💻 Asignación a técnicos
- 📊 Seguimiento en tiempo real
- ✅ Cierre de incidencias
- 📈 Generación de reportes e indicadores
- 🔐 Gestión de usuarios y credenciales

---

## 🧑‍💼 Usuarios del Sistema
- 👤 Usuario (reporta incidencias)
- 🛠️ Soporte técnico (gestiona incidencias)
- ⚙️ Administrador (control total del sistema)

---

## 🏗️ Tecnologías Utilizadas
- 💻 Frontend: HTML, CSS (Tailwind CSS)
- ⚙️ Backend: PHP
- 🗄️ Base de datos: SQL Server
- 🔐 Seguridad: HTTPS, autenticación por roles
- 🔄 Metodología: SCRUM

---

## 📊 Arquitectura del Sistema
El sistema está compuesto por los siguientes módulos:

- 📝 Registro de incidencias  
- 🔧 Gestión de tickets  
- 📡 Seguimiento  
- 📊 Reportes y métricas  

---

## ⚙️ Requisitos del Sistema

### Funcionales
- Crear, modificar, consultar y cerrar incidencias  
- Asignar tickets a técnicos  
- Visualizar reportes  

### No Funcionales
- ⏱️ Tiempo de respuesta menor a 3 segundos  
- 🔐 Seguridad mediante autenticación  
- 🌐 Disponibilidad mínima del 95%  
- 💻 Compatible con navegadores modernos  

---

## 📅 Metodología de Desarrollo
Se utilizó la metodología ágil **SCRUM**, trabajando por sprints de 2 semanas, con entregas periódicas y seguimiento continuo.

---

## 📌 Alcance del Proyecto
- Implementación de una mesa de ayuda interna  
- Automatización del soporte técnico  
- Generación de métricas para toma de decisiones  

---

## ⚠️ Problema que Soluciona
Actualmente la empresa gestiona incidencias de forma manual (llamadas, correos, mensajes), lo que genera:
- ❌ Falta de trazabilidad  
- ⏳ Retrasos en la atención  
- 📉 Baja eficiencia  

Este sistema soluciona estos problemas mediante una plataforma centralizada.

- 📊 Mejor control y seguimiento
